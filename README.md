# Testimonial Management API

RESTful API для платформы сбора, управления и анализа видеоотзывов (testimonials). Позволяет компаниям регистрироваться, собирать отзывы клиентов, управлять их жизненным циклом, отслеживать публикации в социальных сетях и получать аналитическую информацию.

---

# Технологический стек

- **Runtime:** Node.js v16+
- **Framework:** Express.js 4
- **Database:** MongoDB + Mongoose
- **Authentication:** JWT
- **Validation:** Joi
- **Testing:** Jest + Supertest
- **Security:** Helmet, CORS, express-rate-limit

---

# Установка и запуск

## 1. Клонирование репозитория

```bash
git clone <url_вашего_репозитория>
cd TestimonialManagementAPI
```

## 2. Установка зависимостей

```bash
npm install
```

## 3. Создание файла `.env`

Создайте в корне проекта файл `.env` со следующим содержимым:

```env
PORT=3000
MONGODB_URI=mongodb://localhost:27017/testimonials_db
JWT_SECRET=your_super_secret_key_here
JWT_EXPIRY=7d
```

## 4. Запуск приложения

Development

```bash
npm run dev
```

Production

```bash
npm start
```

После запуска API будет доступен по адресу:

```
http://localhost:3000
```

Проверить работоспособность сервера можно запросом:

```
GET /health
```

## 8. Security Notes

### CORS Configuration

В production окружении переменная `ALLOWED_ORIGINS` обязательна. Пример:

```env
ALLOWED_ORIGINS=https://yourdomain.com,https://app.yourdomain.com
```

### Rate Limiting

- General API: 100 запросов за 15 минут
- Login: 5 попыток за 1 минуту
- Register: 10 попыток за 1 минуту

Rate limiting автоматически отключается в тестовом окружении.


# Архитектура проекта

Проект построен по **MVC + Service Layer** архитектуре с разделением ответственности между слоями:

```
├── controllers/
│   ├── testimonialController.js  # CRUD operations for testimonials
│   ├── settingsController.js       # Settings management
│   └── analyticsController.js      # Analytics endpoints
├── services/
│   ├── testimonialService.js       # Business logic for testimonials
│   ├── settingsService.js          # Business logic for settings
│   └── analyticsService.js         # Business logic for analytics
├── middleware/
├── models/
├── routes/
├── validators/
├── lib/
│   ├── constants.js
│   └── helpers.js                  # Response helpers (success/failure)
├── tests/
├── app.js
├── server.js
└── README.md
```

Такой подход делает код более читаемым, упрощает поддержку и облегчает дальнейшее масштабирование проекта.

# Основные архитектурные решения

### MVC-подобная структура

Логика приложения разделена на модели, контроллеры и маршруты. Это позволяет избежать дублирования кода и делает проект проще для поддержки.

### Soft Delete

При удалении отзыв не удаляется из базы данных физически.

Вместо этого документ помечается:

```javascript
isDeleted: true
deletedAt: Date
```

Такой подход позволяет сохранить историю данных и при необходимости восстановить информацию.

### Изоляция данных пользователей

Каждый запрос проверяет владельца ресурса по `userId`, полученному из JWT.

Это исключает возможность:

* просмотра чужих отзывов;
* изменения чужих настроек;
* удаления чужих данных.

### Единый формат ответов

Все эндпоинты возвращают ответы в едином формате:

```json
{
  "code": 200,
  "status": "success",
  "message": "Operation completed successfully",
  "data": {}
}
```

Это значительно упрощает интеграцию с frontend-приложением.

### MongoDB Aggregation Pipeline

Для получения аналитики используется Aggregation Pipeline (`$match`, `$facet`, `$group`), что позволяет за один запрос вычислить:

* общее количество отзывов;
* средний рейтинг (рассчитывается только по отзывам с указанным рейтингом);
* распределение отзывов по статусам;
* статистику за выбранный период (даты возвращаются в ISO-формате).

Такой подход минимизирует количество обращений к базе данных и снижает нагрузку на приложение.

## Testimonials

| Метод  | Endpoint                                  | Описание                                                        |
| ------ | ----------------------------------------- | --------------------------------------------------------------- |
| POST   | `/api/testimonials`                       | Создать отзыв                                                   |
| GET    | `/api/testimonials`                       | Получить список отзывов (с пагинацией и фильтрацией по статусу) |
| GET    | `/api/testimonials/:testimonialId`        | Получить отзыв по ID                                            |
| PUT    | `/api/testimonials/:testimonialId`        | Обновить текст и рейтинг                                        |
| PATCH  | `/api/testimonials/:testimonialId/status` | Изменить статус (с проверкой допустимых переходов)              |
| POST   | `/api/testimonials/:testimonialId/share`  | Добавить каналы публикации (авто-переход в "shared" при "completed") |
| DELETE | `/api/testimonials/:testimonialId`        | Мягкое удаление отзыва                                          |

# Ответы на вопросы

## Сколько времени заняла разработка?

На реализацию основного функционала, настройку базы данных, разработку бизнес-логики, обработку статусов и подготовку документации было затрачено примерно **8–10 часов** чистого времени.

---

## Какие бонусные задачи выполнены?

### Rate Limiting

Реализована защита эндпоинтов авторизации с помощью `express-rate-limit`.

Это предотвращает brute-force атаки, ограничивая количество запросов с одного IP-адреса.

### Service Layer

Реализован отдельный слой сервисов для бизнес-логики:
- `testimonialService` — логика работы с отзывами (создание, обновление, статусы, публикация)
- `settingsService` — логика работы с настройками
- `analyticsService` — логика получения аналитики

### Response Helpers

Созданы утилиты `success()` и `failure()` для унификации форматов ответов и устранения дублирования кода.

### UUID Validation

Добавлена валидация формата `testimonialId` (UUID v4) для параметров маршрутов, что предотвращает передачу некорректных данных в MongoDB.

### Security Tests

Добавлены тесты на попытку доступа к чужим данным (unauthorized access tests).

## Известные ограничения

- **In-memory MongoDB** используется для тестов, что ограничивает тестирование производительностью
- **Нет реальной отправки** отзывов по email/SMS — функционал имитируется
- **Нет реальной загрузки видео** — поле `videoUrl` принимает только URL
- **Агрегация аналитики** вычисляет средний рейтинг только по отзывам с указанным рейтингом
- **Нет кэширования** — аналитика вычисляется при каждом запросе
